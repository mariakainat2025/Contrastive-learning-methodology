----------------------------------------------------------------------
  TIER 1 — EDR CORE (auditd / eBPF / Sysmon)
----------------------------------------------------------------------

  auditd:SYSCALL                [144 techniques]
    Syscall-level events — process, file, network, socket (highest
    EDR coverage)

  auditd:EXECVE                 [ 16 techniques]
    Process execution with full command-line arguments

  auditd:PATH                   [  7 techniques]
    File path arguments from syscalls (open, rename, unlink)

  auditd:FILE                   [  3 techniques]
    File-level audit events

  auditd:PROCTITLE              [  3 techniques]
    Full process title/command line

  auditd:MMAP                   [  1 techniques]
    Memory-mapped file events

  auditd:CONFIG_CHANGE          [  2 techniques]
    Audit configuration change events

  auditd:AUTH                   [  1 techniques]
    Authentication events via auditd

  auditd:USER_LOGIN             [  1 techniques]
    User login events via auditd

  linux:Sysmon                  [  6 techniques]
    Sysmon for Linux — process create, file, network events
    (EDR-equivalent)

  ebpf:syscalls                 [  0 techniques]
    eBPF syscall tracepoints — modern EDR kernel hook

  ebpf:tracepoints              [  0 techniques]
    eBPF kernel tracepoints

----------------------------------------------------------------------
  RECOMMENDATION FOR YOUR PIPELINE
----------------------------------------------------------------------

  Your THEIA dataset is provenance-graph data captured at the
  kernel syscall level — equivalent to auditd:SYSCALL.

  Priority log sources to configure in your EDR:

    1. auditd:SYSCALL    — 144 techniques  (MUST HAVE)
    2. NSM:Flow          —  35 techniques  (network context)
    3. linux:syslog      —  41 techniques  (auth + daemon events)
    4. auditd:EXECVE     —  16 techniques  (process execution detail)
    5. linux:osquery     —  14 techniques  (optional: state queries)

  Sources to SKIP (Windows-only, no Linux relevance):
    - WinEventLog:Sysmon, WinEventLog:Security, etc.

======================================================================
  WHAT YOUR THEIA PROVENANCE GRAPH CAPTURES
======================================================================

  EVENT_EXECUTE  → auditd:EXECVE / auditd:SYSCALL (execve)
  EVENT_READ     → auditd:SYSCALL (read/openat)
  EVENT_WRITE    → auditd:SYSCALL (write/openat)
  EVENT_CLONE    → auditd:SYSCALL (clone/fork)
  CONNECT        → auditd:SYSCALL (connect/socket) + NSM:Flow
  RECVFROM       → auditd:SYSCALL (recvfrom) + NSM:Flow
  EVENT_UNLINK   → auditd:SYSCALL (unlink)
  EVENT_MMAP     → auditd:MMAP

  All THEIA events map directly to auditd:SYSCALL — the single
  most important log source for Linux EDR detection.
