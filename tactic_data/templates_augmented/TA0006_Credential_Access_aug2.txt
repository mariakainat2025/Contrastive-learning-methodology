======================================================================
  MITRE ATT&CK — TACTIC: TA0006
======================================================================
  ID         : TA0006
  NAME       : Credential Access

  OBJECTIVE  : The adversary is trying to steal authentication credentials such as usernames and passwords.

----------------------------------------------------------------------
  TECHNIQUES USE
----------------------------------------------------------------------

  Credential Access consists of techniques used to acquire credentials
  including usernames, passwords, and other authentication secrets.
  Common approaches include keylogging, credential dumping, and
  extraction of stored credential material.

----------------------------------------------------------------------
  OUTCOME / TACTIC LINKS
----------------------------------------------------------------------

  Use of valid credentials allows attackers to access systems more
  stealthily, avoid detection, and expand access by creating or using
  additional accounts.

----------------------------------------------------------------------
  LINUX DETECTION STRATEGIES
----------------------------------------------------------------------

  Detect unauthorized changes to /etc/hosts, /etc/resolv.conf, or
  abnormal ARP activity, correlated with unusual network sessions or
  service launches.

  Multiple failed login attempts followed by a successful
  authentication from the same source.

  Detect access to credential storage locations such as /etc/shadow,
  GNOME Keyring, KWallet, and browser credential databases. Monitor
  suspicious file reads or extraction attempts.

  Detect exploitation of PAM or authentication services. Indicators
  include PAM failures, service crashes, or abnormal authentication
  flows followed by unauthorized access.

  Detection of credential harvesting from browser sessions, HTTP
  headers, or token caches in memory. Monitor access to sensitive
  paths such as ~/.mozilla, ~/.config/chromium, and AWS credential
  files.

  Detect access to input devices (/dev/input/*, evdev) by processes
  without GUI context, indicating possible keylogging activity.

  Detect changes to PAM configuration files or installation of
  malicious PAM modules, correlated with execution of unknown
  binaries.

  Detect keylogger behavior via input device access or kernel module
  loading (insmod), or processes reading input streams outside normal
  user interaction.

  Monitor authentication systems for abnormal MFA request patterns or
  repeated login challenges inconsistent with user behavior.

  Correlate promiscuous mode NIC settings with execution of packet
  sniffing tools like tcpdump or tshark from unexpected contexts.

  Detect access to /proc/*/mem or /proc/*/maps targeting authentication
  services, often combined with memory dumping utilities such as gcore
  or gdb.

  Detect access to browser cookie databases or memory inspection via
  ptrace targeting browser or session processes.

  Monitor access to certificate stores and execution of OpenSSL or
  PKCS tools used for credential export or manipulation.

  Detect access to Kerberos or SSSD credential stores indicating
  ticket theft or replay attempts, correlated with abnormal login
  behavior.

  Detect unauthorized access to sensitive files such as
  .bash_history, /etc/shadow, or SSH private key directories.
