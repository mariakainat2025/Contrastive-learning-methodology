======================================================================
  MITRE ATT&CK — TACTIC: TA0011
======================================================================
  ID         : TA0011
  NAME       : Command and Control

  OBJECTIVE  : The adversary is attempting to communicate with compromised systems in order to control them.

----------------------------------------------------------------------
  TECHNIQUES USE
----------------------------------------------------------------------

  Command and Control consists of techniques used by adversaries to
  maintain communication with systems under their control inside a
  victim network. Attackers often try to blend in with normal,
  expected traffic patterns to avoid detection. There are multiple
  methods to establish C2 channels, with varying levels of stealth
  depending on the target network architecture and defensive controls.

----------------------------------------------------------------------
  LINUX DETECTION STRATEGIES
----------------------------------------------------------------------

  Detection of suspicious curl, wget, or custom socket-based traffic
  using DNS, HTTPS, or IRC-like protocols, often showing beaconing
  behavior or irregular timing intervals.

  Monitor file write activity to removable media mount points (e.g.,
  /media, /run/media) followed by execution or access of the same
  files on another system.

  Detect curl/wget-based payload delivery to /tmp or /var/tmp followed
  by execution, and correlate with IDS/IPS alerts or packet inspection
  showing anomalous content types.

  Detection of outbound traffic encoded using gzip, Base64, or hex
  encoding prior to transmission via curl, wget, or raw sockets,
  typically from non-standard processes.

  Identification of previously unseen or custom userland processes
  generating high-volume HTTP traffic with unusually low response data.

  Monitor audit logs and DNS resolver activity for repeated failed
  lookups or connections to high-entropy or algorithmically generated
  domain names, correlated with process lineage such as Python, bash,
  or unknown daemons.

  Processes like curl, wget, python, socat, or custom binaries
  initiating TLS sessions to uncommon destinations, with indicators
  such as connect() syscalls, dynamic SSL library usage, and persistent
  encrypted traffic from non-network services.

  Detection of outbound communication over non-standard ports or
  covert channels (ICMP, DNS tunneling) originating from processes that
  do not normally generate network traffic.

  Detection of traffic filtering or manipulation via iptables, nftables,
  or proxy rules designed to evade security scanning or monitoring
  tools.

  Shell-based utilities such as curl, wget, or scp initiating external
  connections followed by creation of executable artifacts on disk.

  Detection of staged execution chains where a script downloads data via
  curl/wget, writes it to disk or memory, then triggers a second-stage
  process that initiates a new outbound connection.

  ICMP or raw socket traffic generated by user-space tools such as bash,
  Python, netcat, or hping3, including crafted packets via scapy or
  libpcap.

  Detection of unusual services binding to non-standard ports or
  initiating client connections with mismatched protocol/port behavior.

  Tools such as sshd, socat, or custom binaries used for tunneling,
  reverse shells, or port forwarding, often encapsulating protocols like
  RDP or SMB within SSH or HTTP channels.

  Detection of user-space utilities (socat, ncat, ssh, iptables) used in
  abnormal configurations for reverse shells, tunnels, or cross-host
  communication with unusual external endpoints.

  Detection of RAT execution chains involving persistence via systemd
  and long-lived external communication sessions, sometimes spawning
  interactive shells.

  Detection of port-knocking sequences followed by firewall rule changes
  or newly opened listening services, indicating dynamic C2 activation.

  Detection of outbound HTTPS traffic from scripts or agents to services
  such as Discord, Slack, Dropbox, or cloud APIs in contexts that are
  inconsistent with normal application behavior.
