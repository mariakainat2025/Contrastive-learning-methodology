======================================================================
  MITRE ATT&CK — TACTIC: TA0006
======================================================================
  ID         : TA0006
  NAME       : Credential Access

  OBJECTIVE  : The adversary is attempting to obtain login credentials such as usernames and passwords.

----------------------------------------------------------------------
  TECHNIQUES USE
----------------------------------------------------------------------

  Credential Access includes techniques designed to steal credentials
  such as usernames, passwords, and authentication tokens. Typical
  methods include keylogging, credential dumping, and extraction of
  stored credential data from systems.

----------------------------------------------------------------------
  OUTCOME / TACTIC LINKS
----------------------------------------------------------------------

  Use of legitimate credentials enables adversaries to access systems
  more discreetly, reduce detection chances, and establish additional
  accounts to support their objectives.

----------------------------------------------------------------------
  LINUX DETECTION STRATEGIES
----------------------------------------------------------------------

  Detect unauthorized changes to /etc/hosts, /etc/resolv.conf, or
  abnormal ARP traffic patterns. Correlate file changes with unusual
  network activity or service creation events.

  Sequences of failed authentication attempts followed by a successful
  login from the same IP or user account.

  Detect access to credential storage locations including /etc/shadow,
  GNOME Keyring, KWallet, and browser credential databases. Monitor
  abnormal process access or reads of these files.

  Detect exploitation of PAM or authentication subsystems. Indicators
  include authentication failures, service crashes, or memory errors
  followed by unauthorized successful logins.

  Detection of credential theft via browser sessions, HTTP headers, or
  in-memory token caches. Monitor access to sensitive directories such
  as ~/.mozilla, ~/.config/chromium, and ~/.aws/credentials.

  Detect access to input devices (/dev/input/*, evdev) by processes
  lacking a legitimate GUI session, indicating potential keylogging.

  Detect unauthorized modifications to PAM configuration files or
  installation of malicious PAM modules, correlated with execution of
  suspicious binaries.

  Detect keylogging behavior via input device access, kernel module
  loading (insmod), or non-interactive processes reading input streams.

  Monitor MFA systems for unusual authentication request frequency or
  inconsistent challenge-response behavior.

  Correlate network interface promiscuous mode with execution of
  packet capture tools such as tcpdump or tshark from unexpected
  users or contexts.

  Detect processes accessing /proc/*/mem or /proc/*/maps targeting
  authentication services like sshd or login, often paired with memory
  dumping tools such as gcore or gdb.

  Detect access to browser cookie storage or memory inspection via
  ptrace targeting browser session processes.

  Monitor access to certificate stores and execution of OpenSSL or
  PKCS tools used for credential extraction or modification.

  Detect access to Kerberos or SSSD credential stores indicating
  ticket theft or replay attempts, correlated with abnormal
  authentication activity.

  Detect unauthorized reads of sensitive files such as
  .bash_history, /etc/shadow, or SSH private key directories.
