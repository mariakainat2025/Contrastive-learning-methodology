======================================================================
  MITRE ATT&CK — TACTIC: TA0004
======================================================================
  ID         : TA0004
  NAME       : Privilege Escalation

  OBJECTIVE  : The adversary is attempting to obtain higher-level privileges.

----------------------------------------------------------------------
  TECHNIQUES USE
----------------------------------------------------------------------

  Privilege Escalation consists of techniques that allow attackers to
  gain elevated permissions on a system or network. Adversaries may
  initially operate with limited access but need higher privileges to
  complete their objectives. Common approaches include exploiting
  vulnerabilities, misconfigurations, and system design weaknesses.
  Elevated access can include SYSTEM/root privileges, local
  administrator roles, or user accounts with administrative-level
  capabilities.

----------------------------------------------------------------------
  OUTCOME / TACTIC LINKS
----------------------------------------------------------------------

  These techniques often overlap with Persistence mechanisms, since
  features that maintain access may also run under elevated
  privileges.

----------------------------------------------------------------------
  LINUX DETECTION STRATEGIES
----------------------------------------------------------------------

  Monitor audit logs for changes to setuid/setgid bits, mismatches
  between UID and EUID indicating privilege escalation, and execution
  of privileged binaries by standard users.

  Use of built-in tools or scripts (e.g., `usermod`, `passwd`,
  `groupmod`) to modify privileges or escalate user access,
  correlated with authentication and process events.

  Correlation of systemd unit or /etc/init.d modifications with
  unusual process behavior during system startup.

  Detection of changes or execution of shell initialization scripts
  such as .bashrc, .profile, or /etc/profile that may contribute to
  elevated execution paths.

  Detects creation or modification of systemd services, cron jobs
  executed at boot, or writes to /etc/init.d/, including permission
  changes and systemctl activity originating from non-root users.

  Detection of container escape attempts using syscalls such as
  unshare, keyctl, or mount, indicating breakout from namespace
  isolation into the host system.

  Detects auditd or inotify rule changes tracking system files,
  combined with execution of scripts or binaries via cron or systemd
  timers.

  Detects exploitation of setuid binaries or kernel vulnerabilities,
  often combined with access to /proc/kallsyms or /dev/kmem.

  Detects ptrace or memfd-based injection using syscall monitoring
  (ptrace, mmap), especially when targeting running processes with
  suspicious memory manipulation.

  Detects creation or modification of cron jobs or systemd timer
  units executed by unusual users or outside expected schedules.

  Detection of valid account misuse via SSH logins, sudo/su abuse,
  and anomalous service account behavior outside normal baselines.
