======================================================================
  MITRE ATT&CK — TACTIC: TA0006
======================================================================
  ID         : TA0006
  NAME       : Credential Access

  OBJECTIVE  : The adversary is attempting to obtain account credentials such as usernames and passwords.

----------------------------------------------------------------------
  TECHNIQUES USE
----------------------------------------------------------------------

  Credential Access includes techniques used to steal credentials such
  as usernames, passwords, and authentication material. Common methods
  include keylogging, credential dumping, and harvesting stored
  secrets from systems.

----------------------------------------------------------------------
  OUTCOME / TACTIC LINKS
----------------------------------------------------------------------

  Use of legitimate credentials allows adversaries to access systems
  more easily, reduces detection likelihood, and enables creation of
  additional accounts to further their objectives.

----------------------------------------------------------------------
  LINUX DETECTION STRATEGIES
----------------------------------------------------------------------

  Detect unauthorized modifications to /etc/hosts, /etc/resolv.conf,
  or abnormal ARP traffic. Correlate these changes with unexpected
  network connections or new service creation.

  Multiple failed authentication attempts followed by a successful
  login from the same user or IP address.

  Detect access attempts to credential storage locations such as
  /etc/shadow, GNOME Keyring, KWallet, or browser credential
  databases. Monitor suspicious processes reading or copying these
  files.

  Detect exploitation of authentication services or PAM modules.
  Indicators include abnormal PAM failures, authentication service
  crashes, or memory corruption followed by unauthorized successful
  logins.

  Detection of credential theft via browser sessions, curl/wget
  headers, or token caches in memory. Use audit logs to track access
  to sensitive directories such as ~/.mozilla, ~/.config/chromium,
  and ~/.aws/credentials.

  Detect processes accessing input devices like /dev/input/* or evdev
  without a legitimate GUI session, indicating potential keylogging.

  Detect modifications to PAM configuration files or installation of
  unauthorized PAM modules, correlated with execution of suspicious
  binaries.

  Detect keylogging activity via access to /dev/input, loading of
  kernel modules (insmod), or non-interactive user processes reading
  input devices.

  Monitor login systems for abnormal MFA challenge frequency or
  repeated authentication attempts inconsistent with user behavior.

  Correlate network interface promiscuous mode with execution of
  sniffing tools such as tcpdump or tshark from unexpected users.

  Detect processes accessing /proc/*/mem or /proc/*/maps targeting
  authentication services like sshd or login, often combined with
  memory dumping tools such as gcore or gdb.

  Detect access to browser cookie storage files such as Firefox or
  Chrome databases, or reads of process memory via ptrace targeting
  browser sessions.

  Monitor access to certificate directories and execution of OpenSSL
  or PKCS tools used to export or modify certificates.

  Detect access to SSSD or Kerberos credential stores indicating
  ticket theft or replay, correlated with unusual authentication
  activity.

  Detect unauthorized reads of sensitive files such as .bash_history,
  /etc/shadow, or private SSH key directories.
