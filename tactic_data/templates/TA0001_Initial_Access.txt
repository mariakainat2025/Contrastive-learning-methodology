Tactic Name: Initial Access
ATT&CK ID: TA0001

Objective
The adversary gains their first foothold within a network through external entry points including spearphishing, drive-by download, and exploiting public-facing web servers.

Tactical Intent
Adversaries establish initial presence via external interfaces before moving to Execution, Persistence, and Command and Control.

Behavior Characteristics
Initial Access is characterized by an adversary crossing the external-to-internal boundary through three behavioral families:

Exploitation of external-facing services or user interaction channels including drive-by download, spearphishing attachment or link, and public-facing web server exploitation to deliver malicious content or gain unauthorized access.
Abuse of legitimate credentials or trusted relationships including external remote services, valid accounts, and trusted third-party providers to enter through authorized access points.
Use of physical or proximity-based vectors including removable media autorun, rogue hardware devices, wireless network exploitation, or software supply chain compromise to introduce malicious content into the target environment.
These behaviors represent the first unauthorized crossing of the network perimeter.

Observable Indicators

Unexpected process activity in web client, email client, or public-facing service context.
File artifact appearing in temporary location following external network interaction.
Outbound network activity immediately following inbound external connection.
Inbound connection to public-facing service followed by unexpected child process spawning.
Authentication event from atypical source preceding unexpected internal activity.
Device or media connection event preceding unexpected file or process activity.
Web browser or email client executing a native application or component following external network connection.

Log Sources
Detection opportunities exist in process creation logs, network logs for inbound and outbound external communication, authentication logs for unusual remote access, and file system logs for payload drops following external interaction.
