Tactic Name: Initial Access
ATT&CK ID: TA0001

Objective
Adversaries work to achieve their first foothold within a network through external entry points, encompassing spearphishing attacks, drive-by download delivery, and exploitation of vulnerabilities in public-facing web servers.

Tactical Intent
The establishment of initial presence through external interfaces enables adversaries to proceed toward Execution, Persistence, and Command and Control objectives.

Behavior Characteristics
Initial Access is characterized by the adversary's crossing of the external-to-internal boundary, manifesting through three behavioral families:

Abuse of internet-facing services or human interaction channels to deliver malicious payloads or gain unauthorized access to internal resources.
Leveraging of compromised credentials or trusted external relationships to pass through sanctioned network entry points.
Exploitation of physical or wireless proximity vectors to introduce malicious material into the target environment.
Collectively, these behaviors represent the adversary's initial unauthorized crossing of the network perimeter.

Observable Indicators

Unusual process activity emanating from a web browser, email client, or externally reachable service.
A file artifact written to a temporary directory location following communication with an external network resource.
Outbound network activity generated in direct response to a preceding inbound external connection.
An authentication or session creation event from an atypical or unrecognized source preceding unexpected internal system behavior.
A peripheral device or removable media mount event followed by unexpected file creation or process execution activity.
A browser or email process executing a locally installed application or native component after contacting an external network resource.
Credential or form data sent to a second distinct external destination after initial interaction with a phishing or lure site.

Log Sources
Detection of Initial Access relies on process creation logs reflecting unexpected child processes from externally facing applications, network logs for both inbound exploitation attempts and outbound callback connections, authentication logs detecting remote access anomalies, and file system logs recording payload drops following external communication.
