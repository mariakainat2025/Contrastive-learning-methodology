Tactic Name: Execution
ATT&CK ID: TA0002

Objective
The adversary is trying to run malicious code on a local or remote system to achieve operational objectives such as reconnaissance, data theft, or further compromise.

Tactical Intent
Adversaries use this tactic to activate delivered payloads or directly run commands on compromised systems. Execution follows Initial Access and enables subsequent Persistence, Privilege Escalation, and Command and Control activities.

Behavior Characteristics
Execution is characterized by adversary-controlled code running on a system through three behavioral families:

Abuse of legitimate system utilities, interpreters, and APIs to run malicious code directly on compromised systems.
Exploitation of application vulnerabilities or manipulation of execution flow to achieve code execution.
Leverage of automated and remote execution mechanisms to trigger malicious code without direct interaction.
These behaviors indicate attempts to run adversary-controlled code through both interactive and automated execution channels.

Observable Indicators

Unexpected execution event in anomalous process context or parent-child relationship.
Code execution linked to a file artifact in a non-standard or temporary location.
Execution activity occurring outside expected operational context or schedule.
Execution event originating from automated, remote management, or deployment system.

Log Sources
Detection opportunities exist in process creation logs capturing unexpected interpreter or child process spawning, file creation logs identifying malicious drops in user-writable directories, application logs recording abnormal exits followed by process creation, and service and scheduler logs monitoring unauthorized task or service creation.
