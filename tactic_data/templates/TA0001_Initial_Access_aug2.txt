Tactic Name: Initial Access
ATT&CK ID: TA0001

Objective
To establish a first foothold within a target network, adversaries exploit external entry points including spearphishing messages, drive-by download techniques, and weaknesses in internet-facing web servers.

Tactical Intent
By leveraging external interfaces to gain initial presence, adversaries lay the groundwork for subsequent Execution, Persistence, and Command and Control activities.

Behavior Characteristics
Initial Access is distinguished by adversaries crossing the external-to-internal network boundary through three behavioral families:

Targeting of outward-facing services or user interaction channels to deliver malicious content or achieve unauthorized network entry.
Exploitation of valid credentials or trusted relationships to gain entry through permitted access points.
Use of physical proximity or removable media vectors to plant malicious content within the target environment.
These behavioral patterns represent the adversary's first unauthorized breach of the network perimeter.

Observable Indicators

Unexpected process execution occurring within a web client, email client, or public-facing service context.
File artifacts emerging in temporary locations subsequent to external network interaction events.
Outbound network traffic observed immediately following an inbound connection from an external source.
Authentication activity from an unusual or previously unseen source preceding unexpected host-level activity.
New device or media connection events that precede unexpected file or process activity on the host.
Web browser or email client process spawning a native application component after an external network connection.
Outbound data transfer to a second external destination following a prior interaction with an initial external site.

Log Sources
Evidence of Initial Access is commonly found in process creation logs, inbound and outbound network communication logs, authentication logs flagging remote access from unusual origins, and file system logs capturing payload drops subsequent to external network activity.
